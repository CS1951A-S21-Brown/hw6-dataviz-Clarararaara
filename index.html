Q1:

The dataset I picked is the video game data. After inspecting the data, I understand that the dataset had the name of games, platforms, publishers, and sales in different regions. For the first question I need to display the top 10 games of all time or a specific year. In order to address that I made a bar plot of the top 10 games sorted by global sales and the global sale amounts are shown in the x axis. I would like to display all time(1980 to 2020) and in a selected range of the year. Therefore, I added a range slider to select a given range of years that users want to see as well as one single year. As default, the top 10 games from 1980 to 2020 (all times) is shown. If people want to look at a specific year, they can drag the slider handles together focus on one year. The video game may have the same name in different platforms, and sometimes the same game can be popular in different platforms leading to an overlapped bar, so I added a dropdown list to filter the bar based on the selection of platforms. I exclude the platforms that have fewer than 20 records from the selection. The second question asked to show top game genre in each sales region. I decided to use map for visualizing the sales region. There are four regions provided in the data. EU, North America, Japan, and others, I color coded each region on the map to differentiate them (others would be everywhere other than EU, NA and JP). In order to show the top genre, I used a legend to call out the genre name in each region with corresponding colors. Additionally, tooltips are used to show the top genre, when hover over each region on the map. The top genre was filtered and selected by sorting the highest sales in that specific region. The map can also be changes based on the selection of year range and game platform. It is interesting to see how the top genre changes in different game platforms and years. The top genre may be the same in some video game platforms mainly because the selection is too narrowed and probably the same top one game occurs in all regions. The last question asks to choose top publishers in the given genre. In order to do that I used another dropdown for the selection of genre and plotted a donut plot for the top 5 publishers in the given game genre. The corresponding global sales are also shown in the labels on the donut plot. Only the donut plot will be interactive to the genre selection not the map and the bar plot. All three plots can be interactive to the selection of year(s) and game platforms.

(I tried to fit the graph and font size on a 24 in monitor, so I apologize if the graphs look squeezed in a smaller laptop screen.)

D3 Advantages:

* D3 is very powerful in adding interactive elements, meaning that it allows one graph object to contain a lot of information and viewers can gather information from the graph through some interactions. For example, the 3rd graph in my dashboard want to show top publishers of each genre. Instead of having all 12 donut plots (12 genres) on the same page, I use D3 to add interaction for viewer to select genre, so that there is only one plot each time and contains the information he/she need for selecting top publishers based on genre. It helps my visualization concise without losing information.

* D3 is also very flexible with customization, meaning that I can take the control of every element of the graph. It allows me to change emphases of the graph based on the specific data set I have. Therefore, it makes graphs to fit individual dataset instead of having the data fit given setups of the graph. It allows creativity in the visualization

* The size of the graph from d3 is very flexible as well and it can scale based on individual webpages. Anyone with a web browser should be able to see it without adjusting size accordingly. Also, the zoom and transition effects create more accessibility to the graph. For example, I can zoom in the map to see small countries without sacrificing resolutions. It is really helpful to add details in the graph because viewer can zoom in to see. Additionally, the transition effects allows people to visualize changes along with interactions.

D3 Drawbacks:

* D3 may not be a best tool for people who are not familiar with web standards. It requires some time to understand the role of each piece of code and where it shows up in the web. it requires some knowledge to connect the script with the web elements and can be hard to debug. If people just need simple visualization, D3 may require too much work for making simple charts.

* I do not think it is the best tool for exploratory data analysis, because it requires some understanding of the data for making fancy graphs to visualize them. It requires some planning on how you want the data to be visualized and what kind of information you would like to present before you start plotting. When we do EDA, most of the time we do not have an expectation on how the data would look like, and d3 will not be helpful in this case. It takes a lot of time to plot the figure and to switch it to another type if you find it not working.

* In d3, everything starts from scratch so people need to have a sense of relative positions of the graph as well as the scales. Other high end packages or software may have set up the relative scales, but, in d3, you need to position the graph on your own. The position need to be relative to the width and height and the window size. For example, for my map graph, I had set up the scale and position that looks nice on my monitor, but when I open it in the laptop screen, it moves around. Therefore, people use d3 should have a good understanding of the position and scale when they set up the graph, while other packages may set up for you, and you can focus only on the graph itself.
